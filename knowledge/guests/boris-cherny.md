# Boris Cherny

Head of Claude Code at Anthropic. Built Claude Code from a solo hack project into a product that now authors 4% of all GitHub commits globally. Previously at Instagram as a top engineer. Briefly joined Cursor before returning to Anthropic for its safety mission.

## Key Frameworks

### Under-Resource Early Innovation
Claude Code started as a one-person terminal hack because that was the easiest way to build. Being under-resourced forced creative constraints that became product advantages. The terminal form factor kept pace with rapid model improvements when no GUI could.

### Latent Demand + Familiar Context
Bring tools to where people already are. Claude Code succeeded partly by meeting engineers in the terminal, making existing workflows easier while being surprising enough to spark curiosity and adoption.

## Core Ideas

- 100% of Boris's code is written by Claude Code since November 2025; he ships 10-30 PRs daily without editing a single line by hand
- Claude Code's internal post announcing it got two likes initially; nobody thought a terminal-based coding tool would work
- Innovation requires psychological safety to fail, accountability to cut losses, and space to follow threads that feel promising
- The title "software engineer" will start going away, replaced by "builder"; everyone will code, everyone will be a product manager
- Non-technical teams (legal, finance) at Anthropic are getting enormous value from Claude Code for document redlining and BigQuery analysis
- Exponential thinking is deep in Anthropic's DNA; three co-founders were first authors on the scaling laws paper
