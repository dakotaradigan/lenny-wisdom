# Ryan J. Salva

VP of Product at GitHub who incubated and launched GitHub Copilot. Previously led engineering systems at Microsoft. Background in philosophy and aesthetics, which informs his view that software development is fundamentally a creative act.

## Key Frameworks

### Horizon-Based Innovation (Next to EPD)
GitHub ring-fences a team called GitHub Next for second and third horizon work. When an idea shows real customer signal, researchers temporarily transfer to a new EPD squad for knowledge transfer. After the product is operationalized, researchers return to Next. Key insight: bring the people along during the transition rather than just handing off the code.

### The 200ms Rule
For AI code completion to feel magical rather than intrusive, suggestions must appear within approximately 200 milliseconds. This threshold drove fundamental architecture decisions for Copilot.

## Core Ideas

- Copilot originated when OpenAI nearly took down GitHub by cloning all public repositories -- this accidental incident led to a deliberate partnership using the Arctic Code Vault snapshot
- Code languages are easier for AI than natural language because they have constrained semantics
- The inline gray italicized suggestion UX won because staying in the flow is what developers value most
- To build successful moonshot teams: hire smart people, give them creative freedom, do not expect near-term revenue, and watch for medium-confidence signals that connect to real customer problems
