# Inbal Shani

Chief Product Officer at GitHub. Previously a general manager at AWS and Microsoft, and a senior software developer on Amazon Robotics. Background in niche AI and sensor systems before the generative AI era.

## Key Frameworks

### Copilot Design Philosophy
Build AI tools that developers want to use by putting yourself in their shoes. Seamless integration with zero friction is essential -- if developers have to learn a new workflow, they won't adopt. The design must be intuitive enough that adoption feels invisible, not like adding another tool to master.

### AI Productivity Metrics
There is no single metric for AI developer productivity. Combine time-to-value, code quality, security improvements, collaboration speed, and developer happiness. Time alone is misleading -- you can write bad code really fast. Developer happiness is the ultimate north star, composed of many input metrics.

### Problem-First AI Adoption
The biggest mistake companies make is asking "What should we do with AI?" instead of "What problem are we solving, and can AI help?" Work backwards from the customer problem, then identify where AI tools fit. Plastering AI on everything is a recipe for wasted effort.

## Core Ideas

- Generative AI will not replace developers in the near future. Copilot is a copilot, not a pilot. Humans remain essential for creative thinking, innovation, and the judgment that AI cannot replicate.
- Junior developers benefit enormously from AI because it frees them to think about systems and architecture from day one instead of spending years learning basic coding patterns.
- AI-driven testing is massively underhyped. As more code is generated faster, the importance of comprehensive testing -- unit, load, security, penetration -- becomes critical and is a prime area for AI augmentation.
- The future will be hybrid AI: multiple LLM models working together alongside specialized niche AI models for specific domains like autonomous vehicles or aerospace.
